## 3.shell 命令

~~~c
shell是一种命令行解释器，
shell --> "贝壳" 对操作系统起到保护的作用;
用户 和 内核的交互:
  用户在命令行提示符下键入命令文本，开始与Shell进行交互。
  接着，Shell将用户的命令或按键转化成内核所能够理解的指令
  控制操作系统做出响应，直到控制相关硬件设备。
  然后，Shell将输出结果通过Shell提交给用户。

ls 、 cd 、 pwd 、whoami

      b 块设备文件:磁盘(/dev/sda ...)

      p 管道文件:(mkfifo fifo)用于本地进程间的通信；
      l 链接文件:（软链接:相当于windows的快捷方式）
      s 套接字文件: （socket）用于网络间通信；
   rw- rw- r--: 文件的权限 （0664）
      文件的三种权限:
        r --> 读权限  4
        w --> 写权限  2

          用户的权限
          组的权限
          其他权限
   一个权限可以用八进制表示:
       	一个文件的最高权限: 0777
  1: 硬链接数
  jiangcx:用户名
  jiangcx:组名
  903: 文件的大小
  Jul 23 22:48： 文件最后修改的时间
  te1.c 文件名

 路径:
~~~

~~~c
在对文件压缩的时候，压缩文件会把源文件覆盖，解压也是一样;
.gz: 压缩率最低，压缩速度最快
   压缩：
      gzip file.txt ---> file.txt.gz
   解压:
      gunzip file.txt.gz ---> file.txt
.bz2: 压缩率次之， 压缩速度次之
	压缩:
		bzip2 file.txt ---> file.txt.bz2
  解压：
    bunzip2 file.txt.bz2 ----> file.txt
.xz: 压缩率最高，压缩速度最慢
  压缩：
    	xz file.txt ---> file.txt.xz
  解压：
      unxz file.txt.xz ---> file.txt
~~~

## 1.软链接

~~~c
软链接创建：
  	ln -s 源文件 目标文件
 相当于windows的快捷方式;
 一般源文件给绝对路径;
 软链接的文件类型是链接文件;
 当源文件给了绝对路径以后，链接文件可以移动到其他路径下使用;
 源文件给相对路径，链接文件不能移动;
 如果源文件移动了那么，链接文件会失效；
   eg:
		ln -s /home/jiangcx/class/230704/day3/a.out print
                            源文件                链接文件

~~~

## 2.硬链接

~~~c
硬链接的创建: (相当于对文件取别名)
	ln 源文件 目标文件
   硬链接是根据inode创建的文件，硬链接文件和源文件inode号一样;
   当硬链接文件被修改，源文件也会被修改;
   创建硬链接硬链接数会加一;
	 当原文件啊被删除，硬链接数会减一;
   源文件被移动，硬链接数不变， 硬链接正常使用;
~~~

~~~c
  int var1 = 10, var2 = 20;
  int * const p = &var1; // 只能初始化

 3) const 数据类型 * const 指针变量名;
 指针变量既不能修改目标的值，指针变量本身的值也不能修改;
eg:
  int var1 = 10, var2 = 20;
  int * const p = &var1; // 只能初始化
  p = &var2 // flase
~~~

# 4.3  malloc 动态分配内存

~~~c
 #include <stdlib.h>

 void *malloc(size_t size);
{
  功能： 在堆区上申请一片空间；
  参数： size: 表示空间的大小
  返回值: 返回空间首地址，void *类型
   因为工程师可能需要不同类型的空间，所有返回void *类型的地址，
   可以强制类型转换为其他任意类型的地址;
   申请失败返回NULL;
  
}
 void free(void *ptr);
{
  功能: 释放空间
  参数: 空间的首地址
    注意: malloc 申请的空间需要手动释放，如果不释放可能会造成宕机
   把空间释放之后，指向申请的这片空间的指针依旧指向这片空间，需要将
    它指向零地址(待指状态);
}

输入一片数据在堆区上存储；
  
什么是段错误？
  访问了非法内存;
如何规避段错误？
  不能使用野指针；
  malloc 释放了空间之后将指针指向NULL;
~~~

## 5 .二维数组

~~~c
可以由多个一维数组组成：
  int arr[2][3] = {1, 2, 3, 4, 5, 6}
    可以看成由 两个一维数组组成；
  arr[0]
  arr[1]
  
  arr[2][3] --> {arr[0], arr[1]}
元素的访问: 
  arr[0] == *(arr + 0)
  arr[1] == *(arr + 1)
  
  arr[0] 一维数组 ---> {1， 2， 3}
 元素的访问:
	*(arr[0] + 0) == 1 == *(*(arr + 0) + 0) == **arr
  *(arr[0] + 1) == 2 == *(*(arr + 0) + 1)
  *(arr[0] + 2) == 3 == *(*(arr + 0) + 2)
    
    第i行的第j列：
   *(*(arr + i) + j)
  == arr[i][j]
  == (*(arr + i))[j]
  == *(arr[i] + j)
~~~
